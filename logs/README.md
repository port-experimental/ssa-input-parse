# Logs Directory

This directory contains application logs generated by the SSA Inputs Parser.

## Log Files

- `error.log` - Error-level messages only
- `combined.log` - All log messages
- `exceptions.log` - Unhandled exceptions
- `rejections.log` - Unhandled promise rejections

## Log Rotation

Consider implementing log rotation for production use. You can use tools like:
- `winston-daily-rotate-file` for automatic log rotation
- System-level tools like `logrotate` on Linux

## Viewing Logs

```bash
# View error logs
tail -f logs/error.log

# View all logs
tail -f logs/combined.log

# Search logs
grep "ERROR" logs/combined.log
```

## Log Format

Logs are stored in JSON format for easy parsing:

```json
{
  "timestamp": "2024-10-30 10:00:00",
  "level": "info",
  "message": "Processing template",
  "metadata": {
    "inputFile": "input.json"
  }
}
```
